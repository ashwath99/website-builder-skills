## How to Use This File

### Per-task workflow

1. Copy this file into your project folder
2. Fill in the `Value` column with production values for the specific product
3. The agent reads this file at session start and substitutes values wherever `{PLACEHOLDER}` appears in other skill files

### Per-product overrides

If different products use different token values (e.g., different primary colors or fonts), maintain one copy of this file per product:

- `{product-a}-design-tokens.md`
- `{product-b}-design-tokens.md`

The agent reads the product-specific file based on the `{product}` prefix in the prompt.

### What stays locked

These values are structural CSS constants defined in `design-tokens/SKILL.md` and are NOT in this file because they are universal:

- CTA hierarchy order: Primary → Secondary → Tertiary
- `radius-full`: `9999px`
- `radius-none`: `0`
- `shadow-none`: `none`
- `letter-spacing-body`: `normal`
